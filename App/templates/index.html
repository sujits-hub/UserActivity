Documentation page of the Django Project Submitted

Task Given:

  • Design a Django Application with User and ActivityPeriod models
  • Write custom management command to populate the database with dummy data
  • Write an API to serve the data in the json format

You can see the end point here

All Api used in the project are listed below.

  • GET - /activity/get-all-details/
  • GET - /activity/user_list/
  • GET - /activity/user_list// Go to /activity/user_list/ and by clicking a user you can see this page

Implementation

Models

  • User

    • This Model is used to store information about the user.It contains the following attributes:

    • user_id - unique CharField() which contains the id of the user
    • real_name - CharField() which contains the name of the user
    • tz- DateTimeField() which contains the timezone

  • Activity Periods

    • This Model is used to store the session of the user that is activity periods of the user. It contains the following attributes.

    • user - ForeignKey attribute that contains User information
    • start_time - DateTimeField() which contains the start_time of the activity of the user
    • end_time - DateTimeField() which contains the end_time of the activity of the user

Custom Management Command

Dummy data has been populated using BaseCommand. Data is produced using Faker library
